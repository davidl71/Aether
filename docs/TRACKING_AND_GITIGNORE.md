# What We Track vs Ignore

Decisions on which uncommitted paths to add to version control and which to ignore. See `.gitignore` for the actual patterns.

## Decisions (summary)

| Category | Decision | Rationale |
|----------|----------|-----------|
| **Generated code** | **Do not track** | Reproducible at build time; reduces noise and merge conflicts. |
| **Third-party vendor trees** | **Do not track** | Large clones (tws-api, Intel libs) and cache archives; use scripts/docs to obtain. |
| **Build directories** | **Do not track** | Machine-specific build outputs in `build/`, `target/`. |
| **Project source** | **Track** | Rust in `agents/backend/`, Python/scripts where still active, docs, scripts, Ansible, and shared editor config. Legacy `native/` references are historical only. |
| **Project docs** | **Track** | Exarp-go, Ansible, subagents, planning (unless auto-generated reports). |
| **Nix lockfile** | **Track** | `flake.lock` for reproducible Nix builds. |

---

## Do not track (in .gitignore)

- **`native/generated/`** – C++/proto output from build; generated at CMake/build time.
- **`python/generated/`** – Python proto/build output; generated by proto script or build.
- **`web/src/generated/`**, **`web/src/proto/`** – Frontend generated from proto; generate at build or via script.
- **`native/third_party/tws-api/`** – Full TWS API clone; obtain via clone or zip (see README); only build subdirs were ignored before; ignore full tree.
- **`native/third_party/cache/`** – Downloaded archives (e.g. `.tar.gz`, `.zip`); already have root `third_party/cache/`; add native path for clarity.
- **`native/third_party/IntelRDFPMathLib20U4/`** – Intel decimal library (same pattern as existing U2 ignore).
- **`build-ramdisk`** – Symlink to build volume; machine-specific.

---

## Track (add to repo)

- **Agents**
  - `agents/backend/` – Rust backend source (handlers, config, market-data wiring, seed, service/runtime modules).
  - `agents/go/proto/` – Go proto-related code (track unless we later generate and add to ignore).
- **Ansible**
  - `ansible/inventories/`, `ansible/playbooks/development.yml`, `ansible/roles/devtools/vars/`, `ansible/run-dev-setup.sh`.
- **Docs**
  - Historical exarp-go/subagent docs mostly live under `docs/archive/`; use current active docs in `docs/` for present runtime guidance.
- **Legacy native (historical only)**
  - `native/` paths referenced in older notes are historical and should not be
    used as guidance for new active-source tracking decisions unless native code
    is deliberately reintroduced.
- **Python**
  - `python/integration/` – box_spread_models, broker_positions, cache_client, discount_bank_*, ledger_queries, nats_helper.
- **Scripts**
  - `scripts/git-hooks/prepare-commit-msg`, `scripts/git_pull_safe.sh`, `scripts/service.sh`, `scripts/setup_ramdisk.sh`, `scripts/start_all_services.sh`, `scripts/stop_all_services.sh`, `scripts/verify_toolchain.sh`, `scripts/with_nix.sh`.
- **Editor / AI config (shared)**
  - `.cursor/hooks/session-prime.sh`, `.cursor/skills/*.md`, `.opencode/commands/*.md`, `.claude/commands/`.
- **Nix**
  - `flake.lock` – Lockfile for reproducible Nix (already staged when applicable).

Modified files (e.g. `.cursor/commands/exarpauto.md`, `Justfile`, `scripts/*`) and deleted scripts are committed separately when those changes are intended; this doc and .gitignore apply to the “untracked vs track” decision only.
