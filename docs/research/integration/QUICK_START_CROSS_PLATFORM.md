# Quick Start - Cross-Platform Setup

**Date**: 2025-01-27
**One-Command Setup**: Run the automated script on any platform!

---

## 🚀 One-Command Setup

### macOS / Linux

```bash
./scripts/setup_platform_settings.sh --cmake-configure
```

### Windows (PowerShell)

```powershell
.\scripts\setup_platform_settings.ps1 -CmakeConfigure
```

**That's it!** The script will:
1. ✅ Detect your platform and architecture
2. ✅ Find your C++ compiler
3. ✅ Detect include paths
4. ✅ Find Python interpreter
5. ✅ Generate `.vscode/settings.json.user`
6. ✅ Configure CMake preset
7. ✅ Generate `compile_commands.json` for IntelliSense

---

## What Gets Configured

### Auto-Generated Files

1. **`.vscode/settings.json.user`** (gitignored)
   - Platform-specific IntelliSense mode
   - Compiler path
   - Include paths
   - Python interpreter

2. **`build/<platform>/compile_commands.json`**
   - Generated by CMake
   - Used by VS Code for auto-detection

### Platform Detection

The script automatically detects:
- **macOS ARM64**: Apple Silicon Macs
- **macOS x86_64**: Intel Macs
- **Windows x64**: Windows 10/11
- **Linux x64**: Ubuntu/Debian-based systems

## Next Steps After Running Script

1. **Reload VS Code**:
   - `Cmd+Shift+P` (macOS) or `Ctrl+Shift+P` (Windows/Linux)
   - Type: "Developer: Reload Window"

2. **Verify IntelliSense**:
   - Open a C++ file
   - Check that autocomplete works
   - Check that errors are highlighted

3. **Build the project**:
   ```bash
   cmake --build --preset <your-platform-preset>
   ```

## Platform-Specific Presets

After running the script, use these presets:

### macOS ARM64
```bash
cmake --preset macos-arm64-debug
cmake --build --preset macos-arm64-debug
```

### macOS x86_64
```bash
cmake --preset macos-x86_64-debug
cmake --build --preset macos-x86_64-debug
```

### Windows
```bash
cmake --preset windows-x64-debug
cmake --build --preset windows-x64-debug
```

### Linux
```bash
cmake --preset linux-x64-debug
cmake --build --preset linux-x64-debug
```

## Troubleshooting

### Script Fails to Detect Compiler

**macOS**:
```bash
xcode-select --install
```

**Linux**:
```bash
sudo apt-get install build-essential
```

**Windows**:
- Install Visual Studio 2022 (free Community edition)
- Or install MinGW-w64

### Settings Not Applied

1. **Check file exists**: `.vscode/settings.json.user`
2. **Reload VS Code**: `Cmd+Shift+P` → "Developer: Reload Window"
3. **Check VS Code output**: View → Output → Select "C/C++"

### CMake Configuration Fails

1. **Install CMake**:
   ```bash
   # macOS
   brew install cmake ninja

   # Linux
   sudo apt-get install cmake ninja-build

   # Windows
   # Install via Visual Studio Installer
   ```

2. **Check dependencies**:
   ```bash
   ./scripts/check_tws_download.sh
   ```

## Manual Override

If auto-detection doesn't work, edit `.vscode/settings.json.user` manually:

```json
{
  "C_Cpp.default.intelliSenseMode": "macos-clang-arm64",
  "C_Cpp.default.compilerPath": "/path/to/your/compiler",
  "C_Cpp.default.includePath": [
    "${workspaceFolder}/native/include",
    "/custom/include/path"
  ]
}
```
