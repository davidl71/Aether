> **Status update:** The direct Polygon WebSocket feed has been removed from `tui_service/src/main.rs`. Live ticks, candles, and alerts now arrive solely via the backend aggregator (`backend_service/src/main.rs:548-731`) plus `nats_integration`, so the TUI subscribes only to `market-data.*`/`system.alerts`.

### Key Gaps

1. **Charts tab still generates synthetic candles when NATS data is missing.** Although backend publishes `market-data.candle.*`, the UI only renders those snapshots when the stream is healthy and otherwise falls back to procedural candles. `T-1774362544084047000` should lock in regression coverage so the charts keep showing explicit “waiting for data” states instead of silently fabricating candles whenever the backend/NATS path goes offline.

2. **Back-end provider validity still incomplete.** `MarketDataAggregator::process_event` attaches `source` and `source_priority` to `MarketDataEvent`, but there is no UI/metric that explains which provider fulfilled the latest tick/candle or how priorities are resolved. Task `T-1774363484222095000` now focuses on documenting whether backend Polygon/FMP/Yahoo providers are normalized and whether their metadata is fit for read-only trust.

3. **Charts and alerts need regression coverage.** Real candles and alerts now flow through `market-data.candle.*` and `system.alerts`, but `T-1774362544084047000` is required to lock in regression tests so that TUI render states stay truthful whenever those streams stop (e.g., backend disconnected).

### Backend provider priority rules

Providers feed `MarketDataAggregator` (`agents/backend/crates/market_data/src/aggregator/sqlite_aggregator.rs`), which stores the best quote per symbol. Each `MarketDataEvent` includes `source` and `source_priority`, and `process_event()` only writes the new quote when the incoming priority is higher than the stored priority or, on a tie, when the timestamp is fresher. Priorities live in `agents/backend/crates/market_data/src/aggregator/mod.rs`: Nautilus/TWS = 100, Polygon = 70, FMP = 60, Yahoo = 50, Mock = 0. The aggregator also considers quotes stale once `QUOTE_TTL_SECONDS` (30 s) elapses without updates, which can explain why the UI shows the “waiting for data” state even when a provider is configured but currently offline.

Calling code populates those fields. `spawn_broker_market_data_loop` tags IB/TWS quotes as `source_priority` 100 and `source = tws`; `create_provider()` instantiates the configured provider (e.g., `polygon.rs`, `fmp.rs`, `yahoo.rs`), which sets the matching `source`/`source_priority` before `MarketDataIngestor` delivers the event to `MarketDataAggregator`. Downstream, `handle_market_event()` publishes whichever quote wins to NATS and the snapshot, ensuring `market-data.tick.*` and `market-data.candle.*` always represent the most authoritative available source.

`T-1774363484222095000` should now evaluate whether this metadata needs to be surfaced to operators (status badges, log summaries) and whether any provider normalization step is still weakening trust (e.g., ignoring price-quality flags, rewriting spreads, or failing to mark stale providers explicitly). The status bar now emits `SOURCE@PRIORITY (age)` next to the READ-ONLY pill so operators see which provider is currently winning the live tick/candle stream and how fresh it is. Link this analysis to `docs/STORAGE_BOUNDARIES.md` so that the same doc also describes the verification/monitoring tasks now guarding the storage plane (`T-1774369037665139000`, `T-1774369051790207000`, `T-1774369066472142000`, `T-1774369080893711000`, `T-1774369094923761000`, `T-1774369109429277000`).

### Recommended Fixes (market data)

1. Add regression coverage for `market-data.candle.*` and `system.alerts` so Charts and Alerts display explicit waiting states instead of silently generating synthetic candles when the backend/NATS stream pauses (`T-1774362544084047000`).
2. Keep the `IbAdapter` event bridge operational so strategy ticks continue flowing through the shared aggregator and into NATS/snapshot; monitor for lag/dropped events when the broker connection is interrupted.
3. Surface provider metadata (`source`, `source_priority`, staleness) in the UI or logs so operators can validate which backend source is powering the current market-data stream.
